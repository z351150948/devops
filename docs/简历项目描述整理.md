# AgDevOps 简历项目描述整理

本文档整理了 `AgDevOps 运维与交付平台` 在简历、招聘平台、项目汇报中的常用表达，方便直接复制使用。

---

## 1. 简历长版

- 负责设计并实现 `AgDevOps` 运维与交付平台，基于 `Django / DRF / Channels + Vue3 / Element Plus` 搭建统一运维门户，覆盖 `CMDB`、应用发布、容器运维、日志审计、Nginx 管理与 `IaC` 资源编排等能力。
- 主导落地企业级“应用发布”模块，支持 `Docker` 与 `K8s` 双发布模式、审批流、灰度 / 批次发布、回滚、重新执行，并实现发布结果自动同步 `CMDB`、建立应用与目标资源关系。
- 完成 `K8s` 集群运维增强，包括集群摘要、`Pod Terminal`、工作负载扩缩容、`ConfigMap/Secret` 差异预览与历史回滚；补齐 Docker 容器日志、镜像治理等能力，提升平台可观测与可运维性。
- 建立统一 `RBAC` 权限模型，覆盖页面、接口、按钮与 WebSocket 场景；同时完善演示数据、测试用例与 README 展示体系，支持对外演示与项目汇报。

## 2. 简历短版

- 主导应用发布模块建设，支持 `Docker/K8s` 双模式、审批流、灰度 / 批次发布、回滚与 `CMDB` 自动关联。
- 完善 `K8s` 运维、Docker 治理与统一 `RBAC` 权限体系，形成资源治理到发布交付的完整闭环。

## 3. 四段式简历版

### 3.1 项目描述

- 参与设计并持续迭代 `AgDevOps` 运维与交付平台，基于 `Django / DRF / Channels` 与 `Vue 3 / Element Plus` 构建统一运维门户，覆盖 `CMDB`、应用发布、容器运维、日志审计、Nginx 管理、告警中心与 `IaC` 资源编排等核心场景。

### 3.2 技术亮点

- 实现企业级“应用发布”模块，支持 `Docker 环境` 与 `K8s 集群` 双模式发布，具备审批流、灰度 / 批次发布、回滚、重新执行、状态查看与 `CMDB` 自动关联能力。
- 完成 `K8s` 运维增强，包括集群摘要、`Pod Terminal`、工作负载扩缩容、`ConfigMap/Secret` 差异预览与历史回滚。
- 补齐 Docker 治理能力，支持容器日志查看、详情检查、镜像批量删除与悬空镜像清理。
- 建立统一 `RBAC` 权限模型，覆盖页面、接口、按钮与 WebSocket 场景。

### 3.3 个人贡献

- 主导应用发布模块的前后端设计与落地，完成发布单模型、审批流节点、发布策略、回滚重试、CMDB 联动等核心能力建设。
- 完成发布详情、列表筛选、目标展示、业务线 / 环境对齐等产品化页面改造，并与 `CMDB`、`Docker`、`K8s` 模块打通。
- 编写发布链路相关测试、演示数据与文档，完善 README 展示内容，支持项目演示、汇报与交付说明。

### 3.4 难点与解决方案

- **多环境发布统一建模**：将 `Docker` 与 `K8s` 两类目标抽象到统一发布单模型中，同时兼容不同目标字段与状态查询逻辑。
- **发布闭环落地**：将审批、异步执行、回滚、重试、状态回写、CMDB 同步串成完整链路，避免只停留在“提单”层面。
- **真实场景兼容性**：针对 demo 环境和真实环境差异，补充降级与兜底逻辑，保证演示数据可用、详情页不报错。
- **权限一致性**：通过统一 `RBAC` 约束菜单、页面操作和接口访问，确保平台能力扩展时权限模型不失控。

## 4. BOSS / 拉勾 / 猎聘风格

### 4.1 项目名称

- `AgDevOps 运维与交付平台`

### 4.2 项目介绍

- 面向企业内部的统一运维与交付平台，覆盖 `CMDB`、应用发布、容器运维、日志审计、Nginx 管理、告警中心与 `IaC` 资源编排等场景，支持资源治理、应用发布、运维排障与交付编排闭环。

### 4.3 职责描述

- 负责应用发布模块的前后端设计与实现，支持 `Docker / K8s` 双模式发布、审批流、灰度 / 批次发布、回滚、重新执行与状态查看。
- 负责应用发布与 `CMDB` 的联动设计，发布后自动登记应用配置项、发布目标配置项，并建立资源关系。
- 参与 `K8s` 运维能力建设，落地集群摘要、`Pod Terminal`、工作负载扩缩容、配置版本回滚等功能。
- 参与 Docker 治理、统一 `RBAC` 权限接入、演示数据构建与 README / 产品化展示内容完善。

### 4.4 项目亮点

- 将审批、异步执行、发布状态回写、回滚重试、CMDB 同步串成完整发布闭环。
- 支持企业发布平台常见能力，包括审批流、批次发布、灰度发布流程建模与版本治理。
- 具备较强的平台化和产品化特征，不是单一 CRUD 管理后台。

### 4.5 技术栈

- `Django`
- `Django REST framework`
- `Channels`
- `Vue 3`
- `Pinia`
- `Vue Router`
- `Element Plus`
- `Kubernetes Python Client`
- `Docker Compose`
- `Terraform`
- `SQLite`

## 5. 面试展开点

- 发布链路：审批流 → 异步执行 → Docker / K8s 发布 → 状态回写 → CMDB 同步
- 技术难点：SSH 远程执行、K8s API 对接、版本回滚建模、RBAC 权限收敛
- 产品价值：把“资源治理、应用发布、运维排障、交付编排”整合到一个平台里
